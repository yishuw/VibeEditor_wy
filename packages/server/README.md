# @vibeeditor/server

VibeEditor 服务端 —— 基于 Express 的文件操作 API 与 AI Agent 端点。

---

## 架构

```
src/
├── index.ts              # createApp() / startServer() 导出入口
├── run.ts                # 启动入口（读取 app-config.json，调用 startServer）
├── fs/
│   ├── local.ts          # LocalFileSystem — 封装 Node.js fs 模块
│   └── types.ts          # FileEntry 类型定义
├── routes/
│   ├── files.ts          # /api/files/*    — 文件系统 CRUD（10 个端点）
│   ├── agent.ts          # /api/agent/*    — Agent 对话、SSE 流式、编辑应用（3 个端点）
│   ├── workspace.ts      # /api/workspace/* — 工作区管理、Agent 会话持久化（9 个端点）
│   ├── llm.ts            # /api/llm/*      — LLM 提供商 CRUD + 测试（6 个端点）
│   ├── mcp.ts            # /api/mcp/*      — MCP 服务器 CRUD + 测试/工具查询（6 个端点）
│   └── config.ts         # /api/config/*   — JSON 配置文件读写（2 个端点）
├── middleware/
│   ├── auth.ts           # Bearer Token 鉴权中间件（未挂载，需手动引入）
│   └── requestLogger.ts  # HTTP 请求日志中间件
└── workspace/
    └── manager.ts        # WorkspaceManager — 工作区生命周期、AgentRuntime 缓存、会话持久化
```

---

## 启动方式

```bash
# 开发模式（tsx watch，自动重启）
npm run dev -w packages/server

# 生产构建 + 运行
npm run build -w packages/server
node packages/server/dist/run.js

# 通过根目录脚本（自动先构建 agent + core）
npm run dev:server
npm run dev:all      # server + web 同时启动
```

## 配置

### app-config.json（项目根目录，由 run.ts 读取）

```json
{
  "serverPort": 20385,
  "configDir": "./config"
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `serverPort` | `number` | `20385` | 服务端口 |
| `configDir` | `string` | `"./config"` | 配置文件目录（LLM 配置、MCP 配置、workspace 持久化数据） |

### ServerConfig（createApp / startServer 参数）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `port` | `number?` | `20385` | 服务端口 |
| `host` | `string?` | `'0.0.0.0'` | 监听地址 |
| `corsOrigin` | `string?` | `'*'` | CORS 允许的来源 |
| `serveStatic` | `string?` | — | 静态文件目录，设置后托管前端构建产物 |
| `configDir` | `string?` | `'./config'` | 配置文件目录 |

### 环境变量

| 变量 | 默认值 | 优先级 | 说明 |
|------|--------|--------|------|
| `PORT` | — | 最高 | 服务端口 |
| `SERVER_PORT` | — | 次高（fallback） | 服务端口 |
| `SERVE_STATIC` | — | — | 静态文件目录路径 |
| `AUTH_TOKEN` | — | — | Bearer Token（需手动挂载 auth.ts 中间件才生效） |

## 通用约定

### 基础 URL

```
http://localhost:20385
```

### 请求格式

- `Content-Type: application/json`（POST / PUT 请求体为 JSON）
- 请求体大小上限：**50MB**（`express.json({ limit: '50mb' })`）

### 成功响应

所有成功响应的 HTTP 状态码为 `2xx`，Content-Type 为 `application/json`（SSE 端点除外）。

### 错误响应格式

```json
{
  "error": "错误描述信息"
}
```

常见状态码：
| 状态码 | 说明 |
|--------|------|
| `400` | 请求参数缺失或无效 |
| `403` | 路径越权（目录穿越被拦截） |
| `404` | 资源不存在 |
| `500` | 服务端内部错误 |

### 响应头

- `X-Request-Id`: 每个请求自动生成唯一 ID，便于日志追踪

## 公共数据类型

### FileEntry

```typescript
interface FileEntry {
  name: string;           // 文件/目录名
  path: string;           // 相对路径（POSIX 风格，/ 分隔）
  isDirectory: boolean;   // 是否为目录
  size?: number;          // 文件大小（字节）
  modifiedAt?: number;    // 修改时间（Unix 毫秒时间戳）
  children?: FileEntry[]; // 子条目（递归时使用）
}
```

### StoredTab

```typescript
interface StoredTab {
  path: string;          // 文件路径
  cursorLine?: number;   // 光标行号
  cursorColumn?: number; // 光标列号
}
```

### WorkspaceAgentSession

```typescript
interface WorkspaceAgentSession {
  id: string;              // 会话唯一 ID
  name: string;            // 会话名称
  nameAutoGenerated: boolean; // 名称是否为自动生成
  createdAt: number;       // 创建时间（Unix 毫秒时间戳）
  lastMessageAt?: number;  // 最后消息时间
  messages: Array<{
    id: string;
    role: string;          // "user" | "agent" | "system"
    content: string;
    timestamp: number;
    agentId?: string;
  }>;
}
```

### WorkspaceData

```typescript
interface WorkspaceData {
  workspaceId: string;              // 工作区唯一 ID
  rootPath: string;                 // 工作区根目录绝对路径
  rootName: string;                 // 根目录名称
  openTabs: StoredTab[];            // 打开的标签页
  activeTabPath?: string;           // 当前活动标签页路径
  createdAt: number;                // 创建时间（Unix 毫秒时间戳）
  lastOpenedAt: number;             // 最后打开时间
  agentSessions?: WorkspaceAgentSession[]; // Agent 会话列表
  llm?: WorkspaceLLMConfig;         // 工作区级 LLM 配置（预留）
  mcp?: WorkspaceMcpConfig;         // 工作区级 MCP 配置（预留）
}
```

### LLMProvider

```typescript
interface LLMProvider {
  id: string;       // 自动生成的唯一 ID
```

### McpServerConfig

```typescript
type McpServerType = 'stdio' | 'sse' | 'http';

interface McpServerConfig {

  headers?: Record<string, string>; // sse/http 模式：请求头
  sessionId?: string;    // sse 模式：会话 ID

```

| 参数 | 位置 | 类型 | 必填 | 默认值 | 说明 |

| `root` | query | `string` | 否 | `process.cwd()` | 根目录绝对路径 |

### 3. Agent `/api/agent`

所有 Agent 逻辑委托给 `@vibeeditor/agent`，服务端负责路由分派和 Runtime 生命周期管理。

#### `POST /api/agent/chat`

单次 Agent 对话，返回一次完整回复。

**请求体**

```json
{
  "message": "帮我创建一个 Express 服务器",
  "context": {
    "openFiles": [
      { "path": "src/index.ts", "content": "...", "language": "typescript" }
    ],
    "fileTree": "├── src/\n│   └── index.ts\n└── package.json",
    "cursorPosition": { "file": "src/index.ts", "line": 10, "column": 5 }
  },
  "sessionId": "session_123",
  "workspaceId": "ws_abc123",
  "workspaceRoot": "/path/to/project",
  "config": {
    "mode": "build",
    "providerId": "provider_id_here",
    "systemPrompt": "你是一个专业的前端工程师",
    "temperature": 0.7,
    "maxTokens": 4096
  }
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `message` | `string` | 是 | — | 用户消息 |
| `context` | `AgentContext?` | 否 | — | Agent 上下文 |
| `sessionId` | `string?` | 否 | `"default"` | 会话 ID（多轮对话关联） |
| `workspaceId` | `string?` | 否 | — | 工作区 ID（复用已缓存的 Runtime） |
| `workspaceRoot` | `string?` | 否 | — | 工作区根目录 |
| `config` | `object?` | 否 | — | Agent 配置 |
| `config.mode` | `'build' \| 'plan'` | 否 | `"plan"` | `build`：多轮 Agent 循环（调用工具）；`plan`：直接 LLM 对话 |
| `config.providerId` | `string?` | 否 | 活跃提供商 | LLM 提供商 ID |
| `config.systemPrompt` | `string?` | 否 | — | 系统提示词 |
| `config.temperature` | `number?` | 否 | — | 温度参数 |
| `config.maxTokens` | `number?` | 否 | — | 最大 token 数 |

**响应** `200`

```json
{
  "id": "agent_1717430400000",
  "role": "assistant",
  "content": "好的，我来帮你创建一个 Express 服务器...",
  "timestamp": 1717430400000
}
```

#### `POST /api/agent/stream`

SSE 流式 Agent 对话。这是**核心端点**，支持多轮 Agent 循环（`build` 模式）和直接 LLM 流式输出（`plan` 模式）。

**请求体**：与 `/api/agent/chat` 完全一致。

**响应头**

| Header | 值 |
|--------|-----|
| `Content-Type` | `text/event-stream` |
| `Cache-Control` | `no-cache` |
| `Connection` | `keep-alive` |
| `X-Accel-Buffering` | `no` |
| `X-Request-Id` | `<请求唯一ID>` |

**SSE 事件类型**

| 事件类型 | 数据字段 | 说明 |
|----------|----------|------|
| `chunk` | `{ chunk: string }` | LLM 输出的文本片段 |
| `thinking` | `{ thinking: string }` | Agent 思考/推理内容 |
| `tool_start` | `{ tool_start: string }` | 工具调用开始，格式：`🔍 tool_name: description` |
| `tool_end` | `{ tool_end: string }` | 工具调用完成，格式：`tool_name complete` |
| `tool_result` | `{ tool_result: { name: string, content: string } }` | 工具执行结果 |
| `done` | `{ done: true, edits: AgentEditResult[], toolCalls: number }` | 流结束，包含编辑结果和工具调用次数 |
| `error` | `{ error: string }` | 错误信息 |

**心跳保活**：服务端每 15 秒发送 SSE 注释 `: heartbeat` 防止代理超时断开。

**示例 SSE 流**

```
data: {"thinking":"用户想创建一个 Express 服务器..."}

data: {"tool_start":"🔍 read_file: package.json"}

data: {"tool_end":"read_file complete"}

data: {"tool_result":{"name":"read_file","content":"{\n  \"name\": \"my-project\"\n}"}}

data: {"chunk":"好的"}

data: {"chunk":"，我来帮你"}

data: {"chunk":"创建 Express 服务器。"}

data: {"done":true,"edits":[],"toolCalls":1}
```

## SSE 流式协议详解

`POST /api/agent/stream` 是唯一的 SSE 端点。

### 连接建立

1. 客户端发送 POST 请求，Content-Type 为 JSON
2. 服务端设置 SSE 响应头后立即 `flushHeaders()`
3. 如果配置了 MCP 服务器（`build` 模式），先发送 MCP 初始化状态事件

### 事件流

```
: heartbeat                          ← 闲置 15s 后发送的保活注释

data: {"tool_start":"🔍 MCP: 1 server(s), 3 tool(s)}\n\n

data: {"thinking":"分析用户需求..."}\n\n

data: {"tool_start":"🔍 read_file: src/index.ts"}\n\n

data: {"tool_end":"read_file complete"}\n\n

data: {"tool_result":{"name":"read_file","content":"import express..."}}\n\n

data: {"chunk":"根据"}\n\n
data: {"chunk":"你的"}\n\n
data: {"chunk":"代码，"}\n\n

data: {"done":true,"edits":[{"path":"src/app.ts","operation":"create","content":"..."}],"toolCalls":1}\n\n
```

### 客户端接入示例

```typescript
const response = await fetch('http://localhost:20385/api/agent/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    message: '帮我优化这段代码',
    config: { mode: 'build' },
    workspaceId: 'ws_xxx'
  })
});

const reader = response.body!.getReader();
const decoder = new TextDecoder();
let buffer = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  buffer += decoder.decode(value, { stream: true });
  const lines = buffer.split('\n');
  buffer = lines.pop() || '';
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const event = JSON.parse(line.slice(6));
      // 处理事件...
    }
  }
}
```

### 事件处理参考

| 事件 | 处理方式 |
|------|----------|
| `chunk` | 追加到输出区，实时展示 |
| `thinking` | 显示在思考面板（可折叠） |
| `tool_start` | 在工具调用面板显示进度 |
| `tool_end` | 标记工具调用完成 |
| `tool_result` | 显示工具返回结果 |
| `done` | 流结束，`edits` 数组包含可应用的编辑建议；`toolCalls` 为工具调用总次数 |
| `error` | 显示错误信息，之后会跟随 `done` 事件 |
| `: heartbeat` | 忽略（SSE 注释） |

## 中间件

### CORS

- 来源：`config.corsOrigin ?? '*'`
- 允许方法：`GET, POST, PUT, DELETE, OPTIONS`

### JSON 解析

- `express.json({ limit: '50mb' })` — 支持大文件读取请求

### 请求日志（requestLoggerMiddleware）

- 自动生成 `requestId`（优先使用请求头 `X-Request-Id`）
- 通过 `X-Request-Id` 响应头返回给客户端
- 日志级别按状态码：2xx/3xx → info，4xx → warn，5xx → error
- 跳过 `/api/health` 路径

### 静态文件托管

- 仅在设置 `SERVE_STATIC` 环境变量时激活
- SPA 路由回退：所有非 API 路径返回 `index.html`

### 认证中间件

- `middleware/auth.ts` 实现了 Bearer Token 验证
- **未引入 `index.ts`**，设置 `AUTH_TOKEN` 环境变量不会生效
- 如需启用，在 `createApp()` 中手动挂载

## 注意事项

- **`run.ts` 是实际启动入口**，读取根目录 `app-config.json` 获取端口和配置目录；`createApp()` / `startServer()` 也可作为库被其他程序引用。
- **路径安全**：`/api/files/*` 全部受目录穿越保护；`/api/config/*` 同样受保护；`/api/workspace/browse` 无沙箱限制，后续可通过配置文件限制浏览范围。
- **Runtime 缓存**：通过 `workspaceId` 参数可在多次请求间复用 `AgentRuntime`（含 MCP 连接），避免重复初始化开销。
- **Agent 会话持久化**：Agent 对话历史随工作区数据持久化到 `.vibeeditor/workspace.json`，关闭后重新打开可恢复。
- **MCP 配置热更新**：`/api/agent/stream` 和 `/api/agent/chat` 在 `workspaceId` 存在时会重新从磁盘加载 MCP 配置，新增/启用的服务器自动生效。
